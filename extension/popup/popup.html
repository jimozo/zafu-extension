        How it works

        What Zafu Does

        Zafu is recipient memory for stablecoin sends. It organizes who you pay, which route they use, and what Transfer Check can confirm before an address reaches a wallet, exchange, or dapp field.

        See Zafu in action →

        Recipient Confidence

        Every trusted recipient shows one state, computed from your local send history: Known recipient (you confirmed a send here), Checked route (history synced from another device), Needs test (send a small test first), Saved instruction (partial route saved), or Review (mismatch, stale, or lookalike — check before sending). Marking a send flips Needs test → Known recipient.

        Transfer Check

        Every paste runs through 5 layers in order. First match wins.

        1. Clipboard guard — pasted address differs from the last same-chain browser copy → Possible Clipboard Mismatch review. No network needed.
        2. Your history — trusted (you sent to) and suspicious (sent to you uninvited) addresses from your wallet transactions. Fully local.
        3. Curated blocklist — 10,000+ confirmed scam addresses bundled and refreshed weekly from ScamSniffer. No network needed.
        4. Community warnings — crowd-sourced attacker addresses flagged by Zafu users. Fetched on startup, updated every 24h.
        5. GoPlus Security — real-time on-chain reputation API for EVM addresses. Last-resort check when all local layers return unknown.

        Getting Started

        1. Wallets → Add your public wallet address. Zafu cannot connect to it or move funds.
        2. Zafu scans public history: recipients you sent to become Trusted; unexpected inbound or dust addresses stay Suspicious.
        3. Mark repeat recipients as Favorites and run Intel on important contacts before high-value sends.
        4. Zafu runs automatically on supported sites — just paste as normal.

        What You'll See When You Paste

        • Green banner — Trusted
          Address is in your history. Auto-confirms in 2 seconds.
        • Warning modal — Suspicious
          In your history but you never sent to it. Likely airdrop or spam token.
        • Confirm modal — Unknown
          Never seen before. Verify manually before proceeding.
        • Danger — Poisoning
          Nearly identical to a trusted address. A common scam — do not proceed.
        • Danger — Clipboard hijacked
          What you copied ≠ what's about to paste. Malware likely.
        • Blocked — Known scam
          Flagged in our blocklist. Cannot be overridden.
        • 🚫
          Verified Malicious
          Team-curated or independently confirmed threat. Bypassable only with explicit override.
        • 🚩
          Community Reported
          Reported by Zafu users or automated poisoning signals. High risk, but not team-confirmed.

        Transfer Check Evidence

        Shows observed, checked, warned, and not-checked transfer evidence after Zafu runs detection: asset and network hints when visible, copied-address match, source evidence, threat signals, and the full address before paste. When the address matches a saved recipient, it names them and reminds you of the route. You can tune the help level in Settings.

        Address Intel

        Run Intel on a contact to review identity and reputation with your own explorer keys: EOA vs contract, verified status, known labels, balance, and risk signals. Intel is local — keys stay on this device. Open the full view to work through contacts by review status.

        Trusted vs. Suspicious

        Trusted — addresses you have actively sent funds or tokens to. These carry a confidence state (Known recipient, Checked route, and so on) from your local send history and sync provenance.

        Suspicious — addresses in your history you never initiated a send to (airdrops, dust attacks, inbound transfers). These trigger a warning modal. Can be promoted to trusted after labelling.

        Network Mode

        No advertising tracking or error reporting. Optional Network Mode shares anonymous aggregate counts only, never addresses, labels, clipboard text, chat text, URLs, balances, transaction hashes, Google ID, or email. Your labels, notes, and trusted contacts stay local unless you choose Google Sign-In for address-book backup.

        Community reports are separate from Network Mode. Manual reports and opt-in automatic contribution can share suspected attacker addresses and signal types with a random install ID. Your wallet, contacts, labels, notes, clipboard text, chat text, and URLs are not submitted.

        Block Explorer API Keys

        Free Etherscan key. Without one, fetches may rate-limit at ~10k tx. One key covers all EVM chains (Ethereum, Polygon, Arbitrum, Base, Optimism, BNB). Solscan key is optional.

        Etherscan (EVM)

        1. Open etherscan.io/apidashboard and sign in (free account).
        2. Click Add to create a new API key. Name it anything (e.g. "Zafu").
        3. Copy the key and paste it in the Etherscan field above.

        Free tier: 5 calls/sec, 100,000 calls/day. Zafu only sends your public wallet address to the explorer.

        Solscan (Solana)

        1. Open pro-api.solscan.io and create a free account.
        2. Go to Dashboard → API Keys and click New Key.
        3. Copy the key and paste it in the Solscan field above.

        Free tier: ~10 req/s. Zafu only sends your public wallet address to Solscan.

        TronScan (TRON)

        1. Open tronscan.org API Keys and sign in (free account).
        2. Click Add to create a new API key. Name it anything (e.g. "Zafu").
        3. Copy the key and paste it in the TRON field above.

        TronScan now requires an API key for requests. Zafu only sends your public wallet address to TronScan.

        Clear all local data

        Resets wallets, contacts, settings, local metrics, cached community data, pending reports, account state, and session data on this device. It does not delete already-submitted community reports or server-side backups synced while signed in.
